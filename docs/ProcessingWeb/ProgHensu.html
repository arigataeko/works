変数

変数は、データを保持するためのメモリ上の「入れ物」のようなものです。 変数には名前がついていて，そこに入れた値を後で参照したり，変更したりできます。 変数を使うには，次のように変数の型と名前を記述して、｢変数の宣言」をします。 
 変数の型　変数名；
変数名は，英数字，アンダースコア(_)からなる文字列で，名前の先頭は数字以外の文字にします。名前の長さは任意です。ただし、Javaやprocessingがキーワードとして使う文字列(予約語)は使えません。
変数の型は、その変数に入れるデータの型を指定するもので、基本データ型と呼ばれる型を指定する場合と、クラス名を指定する場合があります。データがクラスのオブジェクトである時、データ型にクラス名を指定します。 
・基本データ型
    整数   int     【変数宣言の例】int x;
    実数   float   【変数宣言の例】float abc;
    真偽値 boolean 【変数宣言の例】boolean state;

・クラス名
    String         【変数宣言の例】String str;　Stringは文字列のためのクラス
同じデータ型の変数を複数宣言する場合、 
    float x;
    float y;
と書く代わりに、次のように変数名をカンマでつなげ、データ型を省略することができます。 
    float x, y;
変数を宣言すると、デフォルトの値が代入されます。int、floatはゼロ、booleanはfalse、クラスの場合はnull(何も入っていないことを示す特別な値)です。宣言すると同時に値を入れる(代入)することができます。これを初期化といいます。 
    int a = 100； //int型(整数)の変数aを宣言し、100で初期化
この例のように、整数が入る基本データ型の変数の場合、メモリ上の変数の場所にはその数値そのものが格納されます。
一方、オブジェクトが入る変数の場合は、異なります。変数に値を代入する前に、メモリ上にオブジェクトが作られているはずで、変数に格納されるのは、そのオブジェクトがあるメモリ上のアドレス(ポインタとも言う)です。そのため、オブジェクトが入る変数のデータ型のことを参照データ型と呼びます。

定数

プログラムの中に書かれる固定の値が定数で、リテラルとも呼びます。
・基本データ型
    整数intの定数は、例えば10や2013
    実数floatの定数は、例えば12.3や10.123
    真偽値booleanの定数は、trueまたはfalse

・クラス名
    Stringクラスの定数は"abc"や"kyoto"のようにダブルクォーテーションで囲んで表す。
    (文字列定数)

文

プログラムに記述する命令の単位を「文」と呼びます。セミコロンによって文の区切りを表します。
次のような処理をする文があります。

  • ・計算をして、式の値を変数に入れる(代入する)：(算術演算子参照) 
     【例】 a = 1 + 20；
  • ・関数を実行する： 
     【例】line(100,100,150,200)；
  • ・オブジェクトを作って変数に入れる：(クラスとオブジェクト参照) 
     【例】 変数 = new クラス名(引数)；

    • 算術演算子

    算術的な計算をするには、次の算術演算子を使います。

    演算子   意味
    + 足し算
    - 引き算
    * 掛け算
    / 割り算
    % 割り算の余り　例：x%3
    ++ 1を加える　　　例：i++
    -- 1を引く　　　　　例：i--
    例えば、aの中の値を３倍を計算して、結果を変数xに保存するには、次のように書きます。「a*3」という式の値は300で、その値がxに代入されます。 
    int a = 100；
int x = a * 3;
    %は余りを計算する演算子で、次の文でaの値は１になります。 
    int x = 10;
int a = x%3;

    +演算子は足し算をする以外に、特別な働きをします。文字列に対して+演算子を使うと、2つの文字列をつなげます。例えば、 "Sensory" + "Vision" の演算結果は、"SensoryVision"になります。processingでprintln("Value: " + x);と書くと、"Value: 数値"が、画面下部に表示されます。数値は変数ｘの値です。

    コメント

    プログラムに関する説明文をコメントと呼びます。コメントをプログラム内に書いておくと、どのような処理をさせているかが、すぐにわかるので、他の人のためにも自分のためにも便利です。コメントの書き方には次の2種類があります。

    • ・//の後ろ行末までがコメントとなる。
    • ・/＊と＊/ではさんだ部分がコメントとなる。
